# ⚛️ Фронтенд с нуля — Модуль «React»

Учебный проект курса **«Фронтенд с нуля»**. Модуль посвящён библиотеке **React**: от первого компонента до маршрутизации между страницами.

Проект собран на **Vite + React 19** и построен как «живой конспект» — каждый компонент разбирает одну тему, а код снабжён подробными комментариями по шагам (ШАГ 1, ШАГ 2, …).

---

## 🎯 Для чего этот проект

- 📚 **Учебное пособие.** Один репозиторий, в котором собраны все базовые темы React в рабочем виде — можно запустить, потыкать, сломать и починить.
- 🧩 **Пример за примером.** Каждая тема живёт в своём компоненте, поэтому легко найти нужное и не запутаться.
- 🗣 **Комментарии простым языком.** В коде объясняется не только *что* делаем, но и *почему* именно так.
- 🏗 **Реальная структура.** Проект похож на настоящее приложение: страницы, общий макет, шапка/подвал, темизация, переключение языка.

> Цель модуля — чтобы студент после прохождения умел самостоятельно собрать SPA-приложение на React с несколькими страницами и общим состоянием.

---

## 🧠 Какие темы разбираем

| Тема | Где смотреть | Что учим |
|------|--------------|----------|
| **Компоненты и JSX** | `src/App.jsx`, `components/*` | Как React делит интерфейс на переиспользуемые кусочки |
| **Props** | `components/Button.jsx` | Передача данных из родителя в дочерний компонент |
| **useState** | `components/Counter.jsx` | Состояние компонента и перерисовка при его изменении |
| **useRef** | `components/RefDemo.jsx` | Прямой доступ к DOM и хранение значений без ре-рендера |
| **useEffect** | `src/App.jsx` | Побочные эффекты после рендера (работа с `document.body`) |
| **Context API** | `store/ThemeContext.jsx`, `store/LanguageContext.jsx` | Общее состояние без «прокидывания пропсов» через все уровни |
| **Темизация (светлая/тёмная)** | `components/Header.jsx`, `index.css` | Практика Context: переключение темы всего приложения |
| **Мультиязычность (ru/tj/en)** | `LanguageContext.jsx` | Практика Context: переключение языка интерфейса |
| **Роутинг (react-router-dom)** | `src/App.jsx`, `main-layout/Layout.jsx`, `components/RouterGuide.jsx` | Несколько страниц, общий макет через `<Outlet/>`, навигация |

🔜 **Следующий урок:** управление состоянием через **Zustand**.

## 🗺 Страницы приложения

- `/` — главная (`HomePage`)
- `/about` — о нас (`AboutPage`)
- `/contact` — контакты (`ContactPage`)
- `/router` — гайд по роутингу (`RouterGuide`)

Все страницы отрисовываются внутри общего макета `Layout` (шапка + контент + подвал).

## 📁 Структура проекта

```
my-app/
├── index.html
├── vite.config.js
└── src/
    ├── main.jsx              # точка входа, BrowserRouter
    ├── App.jsx               # каркас: провайдеры + роуты
    ├── main-layout/
    │   └── Layout.jsx        # общий макет (Header + Outlet + Footer)
    ├── components/           # компоненты и учебные страницы
    │   ├── Header.jsx        # шапка, переключатели темы и языка
    │   ├── Footer.jsx
    │   ├── Counter.jsx       # демо useState
    │   ├── RefDemo.jsx       # демо useRef
    │   ├── Button.jsx        # демо props
    │   ├── HomePage.jsx / AboutPage.jsx / ContactPage.jsx
    │   └── RouterGuide.jsx   # объяснение роутинга
    └── store/                # React Context
        ├── ThemeContext.jsx
        └── LanguageContext.jsx
```

## 🚀 Запуск проекта

Требуется установленный [Node.js](https://nodejs.org) (LTS).

```bash
cd my-app
npm install       # установить зависимости
npm run dev       # запустить дев-сервер (http://localhost:5173)
```

Другие команды:

```bash
npm run build     # production-сборка в папку dist/
npm run preview   # локальный просмотр production-сборки
npm run lint      # проверка кода линтером (oxlint)
```

## 👨‍🏫 Как учиться по этому проекту

1. Запусти проект и открой его в браузере.
2. Открывай компоненты **по порядку тем** из таблицы выше.
3. Читай комментарии в коде — они идут по шагам.
4. Меняй код и смотри, что изменилось. **Сломать не страшно — это учебный проект.**
5. После каждой темы попробуй повторить её самостоятельно в чистом компоненте.

Удачи в обучении! 🎓
